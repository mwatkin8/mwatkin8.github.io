Determinantes Sociales App Demo

The app was created for health providers in Lima, Peru, and is therefore in Spanish.

Because this is only a demo, many fields are hard coded and non-modifiable. For example, login will only work with the pre-loaded DNI 12345678-9, new patients won't be saved, edits to patient data won't be saved, additional responses to the social determinants of health collection questionnaire won't be saved/displayed in the summary view, and the Google Maps API key has been removed (so no map will display).

The app was built with the following general workflow:

  1. Patient search and verification: a DNI number is often used as a unique patient identifier in Peru. This initial screen allows users to search for patients who have already been registered. In this demo, only the patient with a DNI of 12345678-9 will return a positive search result.

    • "Confirmar" will navigate the user to app's main summary page with that patient loaded into context.
    • "Incorrecto" will navigate the user to a patient registration page.

  2. Patient registration: basic demographic information is collected for new patients.

    • "Entrar" would create a patient FHIR resource and navigate the user to app's main summary page with that patient loaded into context (no new resources are created in this demo and the same sample patient will be loaded).
    • "Cancelar" will return the user the patient search and verification page.

  3. Accessing the questionnaire: the social determinants collection questionnaire can be accessed by clicking on the "Socioeconomico Familiar" option on the left sidebar of the summary page of the app.

    • "Regresar" will return the user to the patient search and verification page.

  4. The questionnaire: because this is a demo, new responses will not be saved or display on the summary page. Buttons have been added to this demo to allow users to view the Questionnaire and sample QuestionnaireResponse FHIR resources (as JSON objects).

    • "Regresar" will return the user to the patient search and verification page.
    • "Enviar Respuesta" will return the user to the patient search and verification page.
